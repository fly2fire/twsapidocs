Error Handling

Whenever the client application performs a request to the TWS which requires a response from it (i.e. placing an order, requesting market data, subscribing to account updates, etc.), the TWS will produce the desired result as long as the request was made appropriately. When the TWS is unable to process the request properly, it will always return an error message pointing in the mistake's direction.

Error messages sent by the TWS are handled by the IBApi.EWrapper.error method. The IBApi.EWrapper.error event contains the originating request Id (or the orderId in case the error was raised when placing an order), a numeric error code and a brief description of it.
